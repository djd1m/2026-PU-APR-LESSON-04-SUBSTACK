## Дизайн-система

| Элемент | Класс Tailwind | Hex |

| Фон страницы | `bg-gray-950` | #030712 |

## Страницы приложения

### 1. Лендинг (`/`)

Публичная страница для привлечения новых авторов и читателей.

**Структура:**

```
[Шапка: логотип + навигация (Войти / Начать писать)]
[Hero секция]
  - Заголовок: "Создайте свой платный newsletter"
  - Подзаголовок: краткое описание платформы
  - CTA кнопки: "Начать бесплатно" | "Посмотреть примеры"
[Секция возможностей: 3 карточки]
  - Пишите и публикуйте
  - Монетизируйте аудиторию
  - Получайте выплаты
[Секция для авторов: как это работает, 4 шага]
[Footer: ссылки на документы, соцсети]
```

**Мобильная версия:** Навигация сворачивается в бургер-меню. Hero — одна колонка.

---

### 2. Страница входа (`/login`)

**Структура:**
```
[Центрированная форма на bg-gray-900]
  - Заголовок: "Вход в SubStack RU"
  - Email input
  - Password input (с кнопкой показать/скрыть)
  - Кнопка "Войти" (bg-indigo-600)
  - Ссылка "Забыли пароль?"
  - Ссылка "Зарегистрироваться"
```

**Валидация:**
- Email: формат проверяется на blur
- Пароль: минимум 8 символов
- При ошибке API — показывается `text-red-400` под кнопкой

---

### 3. Страница регистрации (`/register`)

**Структура:**
```
[Центрированная форма на bg-gray-900]
  - Заголовок: "Создайте аккаунт"
  - Имя input
  - Email input
  - Password input (с требованиями: 8+ символов)
  - Кнопка "Зарегистрироваться" (bg-indigo-600)
  - Ссылка "Уже есть аккаунт? Войти"
```

После успешной регистрации — автоматический вход и редирект в `/dashboard`.

### 4. Дашборд автора (`/dashboard`)

#### Общий макет дашборда

```
[Верхняя панель (header)]
  - Логотип
  - Название публикации автора
  - Аватар / выход

[Боковое меню (sidebar)]
  - 📊 Аналитика       → /dashboard
  - 📝 Статьи          → /dashboard/articles
  - 👥 Подписчики      → /dashboard/subscribers
  - 💰 Выплаты         → /dashboard/payouts
  - ⚙️  Настройки       → /dashboard/settings

[Основное содержимое (main content)]
```

**Мобильная версия:** Sidebar сворачивается до иконок (icon-only режим, ширина `w-16`). Полное меню раскрывается по клику на иконку.

---

#### 4.1 Обзор / Аналитика (`/dashboard`)

```
[Карточки метрик (4 штуки в ряд)]
  - Всего подписчиков (число + delta за 7 дней)
  - Платных подписчиков (число + delta)
  - Просмотры за месяц (число)
  - Доход за месяц (сумма в ₽)

[График просмотров (7 дней)]
[Таблица последних статей: title | просмотры | дата]
```

---

#### 4.2 Список статей (`/dashboard/articles`)

```
[Кнопка "+ Новая статья" → переход в редактор]
[Таблица статей]
  Колонки: Заголовок | Видимость | Статус | Просмотры | Дата
  Строка: клик → редактор, hover → показать кнопки Редактировать / Удалить
[Пагинация]
```

Статусы статьи:
- `Черновик` — `text-gray-400`
- `Запланирована` — `text-yellow-400`
- `Опубликована` — `text-green-400`

---

#### 4.3 Редактор статей (`/dashboard/editor` или `/dashboard/articles/[id]/edit`)

```
[Верхняя панель редактора]
  - Кнопка "← Назад"
  - Статус автосохранения ("Сохранено 2 мин назад")
  - Кнопки: [Черновик] [Запланировать] [Опубликовать]

[Поле заголовка]
  - Placeholder: "Заголовок статьи..."
  - text-3xl font-bold, без рамки, прозрачный фон

[TipTap Тулбар]
  ┌─────────────────────────────────────────────────────┐
  │ B  I  H1  H2  H3  •  1.  "  <>  — Ссылка  Картинка │
  └─────────────────────────────────────────────────────┘
  B = Жирный (Ctrl+B)
  I = Курсив (Ctrl+I)
  H1-H3 = Заголовки
  • = Маркированный список
  1. = Нумерованный список
  " = Цитата
  <> = Блок кода

[Область редактирования TipTap]
  - min-height: 60vh
  - Прозрачный фон, текст text-gray-100
  - Placeholder: "Начните писать..."

[Нижняя панель]
  - Выбор видимости: [Бесплатно ▼] / [Только подписчики] / [Только платные]
  - Счётчик слов
```

**Автосохранение:** каждые 30 секунд изменений, `PATCH /api/articles/:id`.

### 5. Страница публикации (`/[slug]`)

Публичная страница автора, доступная без регистрации.

```
[Header публикации]
  - Аватар автора (круглый, 64px)
  - Название публикации (text-2xl font-bold)
  - Описание (text-gray-400)
  - Счётчик подписчиков

[Форма подписки]
  - Input email
  - Кнопка "Подписаться бесплатно"
  - Кнопка "Поддержать автора — X ₽/мес" (если настроена платная подписка)

[Список статей]
  Карточка статьи:
  - Заголовок (ссылка)
  - Первые 2-3 строки превью
  - Дата публикации
  - Бейдж "🔒 Платно" если visibility=paid
```

---

### 6. Страница статьи (`/[slug]/[article-slug]`)

```
[Шапка статьи]
  - Название публикации (ссылка на /[slug])
  - Заголовок статьи (text-3xl font-bold)
  - Автор + дата публикации

[Тело статьи (TipTap output)]
  - Полный текст или превью (~200 слов) + paywall блок

[Paywall блок (для платных статей, если не подписан)]
  ┌─────────────────────────────────────────────────────┐
  │  Продолжить чтение                                  │
  │  Оформите платную подписку чтобы читать дальше      │
  │  [Поддержать автора — X ₽/мес]                      │
  │  [Подписаться бесплатно]                            │
  └─────────────────────────────────────────────────────┘
  - Фон: bg-gray-900 с blur-эффектом на текст статьи

[Footer статьи]
  - Ссылки поделиться (Telegram, VK, скопировать ссылку)
  - Кнопка "← Все статьи"
```

## Адаптивность (Responsive Design)

| Брейкпоинт | Изменения |
|------------|-----------|
| `sm` (640px) | — |
| `md` (768px) | Sidebar дашборда появляется |
| `lg` (1024px) | Полный sidebar с текстом; 3-колонная сетка на лендинге |
| `xl` (1280px) | Расширенные отступы |

**Мобильный дашборд:**
```
Sidebar → только иконки (w-16)
Header → гамбургер-меню вместо sidebar
Таблицы → горизонтальная прокрутка
Карточки метрик → 2 в ряд (grid-cols-2)
```

## Компоненты UI (переиспользуемые)

| Компонент | Использование |
|-----------|--------------|
| `<Button variant="primary">` | Основные действия (indigo) |
| `<Button variant="secondary">` | Вторичные действия (gray) |
| `<Button variant="danger">` | Удаление, отмена (red) |
| `<Input>` | Текстовые поля с лейблом и ошибкой |
| `<Badge status="published">` | Статусы статей |
| `<StatsCard>` | Карточки метрик в дашборде |
| `<ArticleCard>` | Превью статьи в списке |
| `<PaywallBlock>` | Блок платного контента |
| `<SubscribeForm>` | Форма подписки на странице публикации |
