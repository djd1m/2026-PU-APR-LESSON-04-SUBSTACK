## Карта сетевых портов

Все порты выбраны нестандартными для избежания конфликтов с другими сервисами и Docker-контейнерами на общем сервере VPS.

| Сервис | Внутренний порт | Внешний порт (host) | Протокол | Назначение |
|--------|-----------------|---------------------|----------|-----------|
| NestJS Backend | 3010 | **3010** | HTTP | REST API |
| Next.js Frontend | 3011 | **3011** | HTTP | Web UI (в dev-режиме) |
| PostgreSQL 16 | 5432 | **5437** | TCP | База данных |
| Redis 7 | 6379 | **6380** | TCP | Кеш, очереди Bull |
| MinIO API | 9000 | **9010** | HTTP | S3-совместимое хранилище |
| MinIO Console | 9001 | **9011** | HTTP | Веб-интерфейс MinIO |

### Правила firewall

В продакшне должны быть открыты только порты 80 и 443 (Caddy). Внутренние порты доступны только с localhost:

```bash
# UFW конфигурация
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP (Caddy, redirect to HTTPS)
ufw allow 443/tcp   # HTTPS (Caddy)
ufw deny 3010/tcp   # Backend — только через Caddy reverse proxy
ufw deny 3011/tcp   # Frontend — только через Caddy reverse proxy
ufw deny 5437/tcp   # PostgreSQL — только внутри Docker сети
ufw deny 6380/tcp   # Redis — только внутри Docker сети
ufw deny 9010/tcp   # MinIO — только внутри Docker сети
ufw enable
```

## Docker-сервисы

### Полный список контейнеров

| Контейнер | Образ | Роль |
|-----------|-------|------|
| `backend` | `node:20-alpine` (custom) | NestJS 10 REST API |
| `frontend` | `node:20-alpine` (custom) | Next.js 15 SSR |
| `postgres` | `postgres:16-alpine` | Основная БД |
| `redis` | `redis:7-alpine` | Кеш + Bull очереди |
| `minio` | `minio/minio:latest` | Файловое хранилище (изображения) |

### Docker-сети

Все контейнеры находятся в одной изолированной Docker-сети `substack-net` типа `bridge`. Прямой доступ к контейнерам снаружи этой сети запрещён (кроме портов, явно опубликованных через `ports:`).

### Постоянные тома (Volumes)

| Том | Контейнер | Назначение |
|-----|-----------|-----------|
| `postgres_data` | postgres | Данные PostgreSQL |
| `redis_data` | redis | Данные Redis (persistence) |
| `minio_data` | minio | Файлы пользователей |

## Соответствие требованиям 152-ФЗ

Федеральный закон № 152-ФЗ «О персональных данных» требует хранения персональных данных российских граждан на серверах в России.

### Чеклист соответствия

| Требование | Статус | Реализация |
|-----------|--------|------------|
| Данные на российских серверах | Выполнено | VPS AdminVPS/HOSTKEY (Россия) |
| PostgreSQL в России | Выполнено | Контейнер postgres на том же VPS |
| Резервные копии в России | Требуется настройка | Локальные бэкапы + российский S3 |
| Email-провайдер: DPA | Требуется | Подписать DPA в Resend |
| Платёжные провайдеры | Выполнено | CloudPayments + YooKassa — российские компании |
| Аналитика | Требуется | Использовать Яндекс.Метрику, НЕ Google Analytics |
| Пользовательское соглашение | Требуется | Разместить на сайте согласие на обработку ПДн |

### Данные, считающиеся персональными (ПДн)

В системе к ПДн относятся:
- Email адрес
- Имя/псевдоним пользователя
- IP-адрес при регистрации
- Реквизиты для выплат (ФИО, ИНН, расчётный счёт)
- Данные о платёжных транзакциях

### Что НЕ хранится (PCI DSS)

- Номера банковских карт
- CVV/CVC коды
- PIN-коды

Все карточные данные обрабатываются исключительно на стороне CloudPayments и YooKassa через iframe/redirect. Платформа хранит только токены провайдеров.

```
GET http://localhost:3010/api/health
```

Настройте Uptime Robot на проверку этого URL каждые 5 минут с уведомлением на email при падении.
